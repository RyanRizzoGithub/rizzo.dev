Generative Trees: An study of procedural nature with P5.js

Introduction

Tree Generator is a procedural art program I developed in JavaScript using the P5.js graphics library. At first glance, it simply draws trees with trunks, branches, leaves, and grass. But beneath the surface lies a recursive system for rendering natural-looking structures, complete with textures, randomized patterns, and user-controlled parameters.

This project explores how recursion, geometry, and randomness can be combined to mimic the complexity of nature — giving each tree a unique, hand-painted feel while remaining algorithmically generated.

How It Works

When the program loads, it sets up a canvas and preloads textures for bark, leaves, and grass. A control panel appears beneath the canvas, allowing users to adjust parameters like canopy size, branch density, leaf count, and trunk width. Each time the "Generate" button is clicked, a new tree is created from scratch with a random seed (which can be locked for reproducibility).

At the core is the TreeGenerator class, which builds the tree in three recursive stages:

  • Stump → The trunk is drawn in segments, sometimes leaning or swelling, to create a natural base.
  • Boughs → Each trunk segment can spawn branching boughs. Recursive subdivision controls how many levels deep the branches grow.
  • Leaves → At the ends of branches, textured leaves are added. Their position and rotation are randomized to avoid repetition.

Surrounding the tree, the grass() function recursively fills the bottom of the canvas with textured blades, grounding the scene in a natural environment.

Key Challenges

Challenge 1: Recursive Tree Growth

The biggest challenge was ensuring the recursion had proper base cases. Without strict depth limits, the tree would recurse infinitely and crash. By carefully capping recursion at each stage (stump, boughs, leaves), I ensured realistic tree shapes without runaway loops.

Challenge 2: Texture Mapping

Using textures for bark and leaves was trickier than expected. Each shape (trunk, branch, or leaf) is rendered as a quad or ellipse with mapped texture coordinates. This gives the tree depth and realism compared to flat colors, but required experimenting with beginShape(), vertex(), and texture() in P5.js’s WEBGL mode.

Challenge 3: Natural Randomness

Randomness drives the uniqueness of each tree, but uncontrolled randomness looks chaotic. To keep trees believable, I constrained random values (like branch angles or leaf positions) within natural ranges, and used sliders so users could tune how extreme the variation should be.

Interactivity

One of my favorite features is the control panel, which allows real-time experimentation. By sliding parameters like “Twig Quantity,” “Leaf Size,” or “Branch Factor,” users can shape trees into anything from dense oaks to sparse pines. The “Seed Lock” button ensures reproducibility, letting artists save specific tree designs for later use.

What Makes It Unique

Unlike static illustrations, this generator produces infinite variations of trees. Every click creates something slightly new — a different curvature in the trunk, a shift in the branches, or a scatter of leaves. It’s a tool not just for coding practice, but for exploring generative art and the beauty of controlled randomness.

Conclusion

The Tree Generator is more than a drawing tool; it’s an experiment in simulating nature with code. By combining recursion, geometry, and texture mapping, I was able to create an interactive system that produces endlessly unique and organic results.

Whether used as a generative art piece, a learning exercise in recursion, or a base for a game environment, this project shows how even simple algorithms can grow into something that feels alive.
